How to connect components to the Raspberry Pi Pico GPIO pins. All digital inputs use internal pull-ups — no external resistors needed.

Buttons

Wire one leg to a digital GPIO pin, the other to GND. The firmware activates the internal pull-up, so the pin reads high when open and low when pressed. Any of the 21 digital pins can be used: GP0–GP1, GP4–GP22.

GPIO pin ←— [switch] —→ GND

Touch Pads

Connect a conductive pad (copper tape, bare PCB, foil) directly to a digital GPIO pin. No connection to GND — the firmware senses capacitance changes when you touch the pad. Uses the same digital pin pool as buttons (GP0–GP1, GP4–GP22). Works best on RP2350.

GPIO pin ←— [conductive pad]

Potentiometers

Wire the outer legs to 3V3 and GND, and the wiper (middle leg) to an analog pin: GP26, GP27, or GP28. Max 3 pots. Values are read as 0–127.

3V3 ←— [pot outer] — [pot wiper] —→ ADC pin
GND ←— [pot outer]

LDR (Light Sensor)

Wire the LDR and a fixed resistor (e.g. 10k) as a voltage divider between 3V3 and GND, with the midpoint going to an analog pin (GP26–GP28). Shares the analog pin pool with pots.

3V3 ←— [LDR] — midpoint —→ ADC pin
GND ←— [10k resistor] — midpoint

Accelerometer

Uses an I2C accelerometer (LIS3DH) on fixed pins: SCL = GP3, SDA = GP2. Power the module from 3V3 and GND. No external pull-ups needed — the firmware enables internal pull-ups on the I2C lines. When the accelerometer is disabled, GP2 and GP3 are not used.

GP3 (Accel SCL) ←—→ SCL
GP2 (Accel SDA) ←—→ SDA
3V3 ←—→ VCC
GND ←—→ GND

Tips

  • GP2 (Accel SDA) and GP3 (Accel SCL) are reserved for the accelerometer when enabled — don't assign them to buttons or touch pads
  • Each pin can only be used by one input — buttons and touch pads share the digital pool
  • Pots and the LDR share the 3 analog pins — plan accordingly
  • Changes take effect immediately on the device for testing — click Save to Flash to persist
  • Use Export Project to back up your configuration as a JSON file

Scale Modes

The scale() function maps a 0–127 position value onto notes of a diatonic scale, spanning about 2.5 octaves from the root note. This makes it easy to assign musically consonant notes to controls.

Mode#Character

Conditionals

a > b ? x : y — if a is greater than b, use x; otherwise use y.

Examples

  • 60 — fixed note: middle C
  • pot0 — note follows potentiometer 0
  • pot0 + 24 — pot value shifted up 2 octaves
  • pot0 > 64 ? 72 : 60 — C5 when pot is above halfway, else C4
  • min(pot0, 100) — pot value capped at 100
  • clamp(pot0, 20, 100) — pot value restricted to 20–100
  • lerp(36, 84, pot0) — pot sweeps from C2 to C6
  • scale(60, pot0) — pot sweeps C4 Lydian scale
  • scale(48, pot0, minor) — pot sweeps C3 natural minor scale

Notes

  • All values are integers in the range 0–127
  • Arithmetic saturates (never wraps around)
  • Division by zero returns 127
  • Expressions compile to a max of 16 bytes of bytecode
  • Parentheses () can be used for grouping
